# WeatherNotes

iOS додаток для створення заміток з автоматичним додаванням погоди.

## Про проєкт

Ідея проста - коли створюєш замітку, додаток автоматично отримує поточну погоду в Києві через OpenWeather API і зберігає її разом з текстом.

## Реалізований функціонал

Список заміток з відображенням температури та іконки погоди
Детальний перегляд замітки
Додавання нової замітки з автоматичним отриманням погоди
Пошук по замітках
Статистика (кількість заміток, середня температура)
Видалення через свайп
Підтримка світлої та темної теми

## Технічний стек

SwiftUI 
MVVM - для розділення логіки та UI
URLSession - мережеві запити
UserDefaults - локальне зберігання (для більшого проєкту використав би CoreData)
Codable - серіалізація моделей

## Архітектура

Проєкт побудований за MVVM патерном:

**Models** - Note та Weather моделі, WeatherResponse для парсингу API відповіді

**Services** - WeatherService для роботи з API, StorageService для збереження даних

**ViewModels** - NotesListViewModel управляє списком та бізнес-логікою, NoteDetailViewModel для екрану деталей

**Views** - NotesListView (головний екран), AddNoteView (створення замітки), NoteDetailView (деталі), NoteRowView (комірка списку), StatsView (статистика)

## Особливості реалізації

API запит виконується асинхронно при створенні замітки. Якщо запит не вдався (немає інтернету наприклад), замітка все одно зберігається але без погоди.

Дані зберігаються в UserDefaults через Codable - для цього додатку достатньо, хоча для production краще використати CoreData або Realm.

Пошук реалізований через фільтрацію масиву в реальному часі.

В StatisticsView показується середня температура з усіх заміток де є погода.
